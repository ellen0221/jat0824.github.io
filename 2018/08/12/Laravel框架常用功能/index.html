文件上传

  • Laravel文件系统是基于Frank de Jonge的Flysystemk扩展包

HTTP异常

有些异常描述来自服务器的HTTP错误码,例如:404(页面未找到)、401(认证失败错误)、500(程序出错)

为了在应用中生成这样的响应，可以使用 abort 辅助函数：

abort(404, ‘响应描述’);

如果你想要自定义404错误页面，创建一个resources/views/errors/404.blade.php文件，该视图文件用于渲染程序返回的所有404错误。需要注意的是，该目录下的视图命名应该和相应的HTTP状态码相匹配。abort函数触发的HttpException异常会以$exception变量的方式传递给视图。

队列

  • Laravel的队列服务为各种不同的后台队列提供了统一的API
  • 允许推迟耗时任务(例如发送邮件)的执行,从而大幅提高web请求速度

配置文件

config/queue.php

主要步骤

  1. 迁移队列需要的数据表

在.env文件中修改: QUEUE_DRIVER=database

命令行输入:

php artisan queue:table //迁移数据表
php artisan migrate //执行迁移

  1. 编写任务类

命令行创建任务类:

php artisan make:job SendEmail

  1. 推送任务到队列
  1. 运行队列监听器

命令行执行:

php artisan queue:listen
//执行成功则显示:
//➜ list php artisan queue:listen
//[2018-08-13 08:31:57] Processing: App\Jobs\SendEmail
//[2018-08-13 08:31:58] Processed: App\Jobs\SendEmail

  1. 处理失败任务

命令行执行: 迁移数据表

php artisan queue:failed-table
php artisan migrate
//队列执行失败后会将任务记录到failed_jobs数据表中

命令行查看失败记录:

php artisan queue:failed // 查看failed_jobs数据表的记录
// 若想重新将失败的任务放入队列则执行:
php artisan queue:retry 1 // 参数为任务在数据表中的id或所有(all)

命令行删除队列中的任务:

php artisan queue:forget 1 // 参数为任务在数据表中的id
//删除所有任务则执行:
php artisan queue:flush
