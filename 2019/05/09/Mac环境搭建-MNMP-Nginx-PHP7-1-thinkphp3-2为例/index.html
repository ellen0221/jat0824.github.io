安装HomeBrew

/usr/bin/ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)”

使用brew doctor检查是否存在冲突，使用brew update && brew upgrade升级

安装Nginx

直接使用brew命令安装

brew install nginx

如果需要使用80端口，需要将nginx加入root组中

sudo cp -v /usr/local/opt/nginx/*.plist /Library/LaunchDaemons/
sudo chown root:wheel /Library/LaunchDaemons/homebrew.mxcl.nginx.plist

启动nginx服务

sudo nginx

测试nginx是否安装成功，默认配置文件监听的是8080端口，对8080端口发起请求

curl -IL http://127.0.0.1:8080

nginx相关命令

sudo nginx //启动nginx
sudo nginx -s reload|reopen|quit //重新加载|重启|退出

安装php-fpm

添加php-fpm源

brew tap homebrew/dupes
brew tap homebrew/php

安装php-fpm

// 在此我们安装php7.1 如果想安装php5.6等只需要将php71改为php56等
// 安装之前可以查看安装选项 brew options homebrew/php/php71
brew install php71 –whitout-apache –with-imap –with-tidy –with-debug –with-pgsql –with-mysql –with-fpm

安装完成后，将php加入$PATH

可设置php-fpm的开机启动

mkdir -p ~/Library/LaunchAgents
ln -sfv /usr/local/opt/php71/homebrew.mxcl.php71.plist ~/Library/LaunchAgents/
launchctl load -w ~/Library/LaunchAgents/homebrew.mxcl.php71.plist

监测php-fpm是否启动成功

lsof -Pni4 | grep LISTEN | grep php

遇到的问题

访问项目的url时显示502

通过查看/usr/local/etc/nginx/logs/error.lognginx的错误日志，来看报502错的原因，发现：connect() to unix:/tmp/php-cgi.sock failed (2: No such file or directory) while connecting to upstream

在nginx站点配置文件里fastcgi_pass的设置是：fastcgi_pass unix:/tmp/php-cgi.sock;，mac下好像是没有的，所以只能使用TCP端口连接fastcgi，即改为：fastcgi_pass 127.0.0.1:9000;

然后再重启nginx服务即可
