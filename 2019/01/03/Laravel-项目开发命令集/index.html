    常用命令

    • 查看laravel版本

      // 进入项目目录后
      php artisan -V

    • 创建laravel项目

      composer create-project –prefer-dist laravel/laravel projectname
      // 安装指定版本
      composer create-project laravel/laravel projectname 5.7.x-dev –prefer-dist

    • 创建数据迁移表

      php artisan make:migration create_staff_table –create=staff
      // –create=tablename 指定的是要创建的数据表名

    • 创建模型

      php artisan make:model Staff -m // -m选项会同时建立一个与模型同名的数据迁移表

    • 创建控制器

      php artisan make:controller StaffController

    • 数据迁移

      php artisan migrate

    • 启动本地服务

      php artisan serve

    • 生成系统内置的脚手架(用户认证)

      php artisan make:auth

    • composer扩展包安装、更新

      composer install
      composer update

    • 生成密钥

      php artisan key:generate

    • php版本切换

      cd /usr/local/bin
      echo $PATH // 查看当前环境变量及优先级（显示在前的先）
      vim ~/.bash_profile // 根据前面输出的环境变量将export的内容修改
      source ~/.bash_profile
      php -v // 查看当前php版本

    laravel web 项目中遇到的各种错误

    安装laravel mix 后，执行 npm run watch-poll 报错：

    • 解决方法：删除 node_modules 目录后重新 yarn

    $ rm -rf node_modules

    $ apt-get install libpng-dev

    vagrant@homestead:~/Code/laravel-shop$ SASS_BINARY_SITE=http://npm.taobao.org/mirrors/node-sass yarn // 项目根目录下安装

    $ npm run watch-poll // 重新执行

    再次报错：

    $ npm install vue-template-compiler

    $ npm run watch-poll // 再次执行，成功！

    用Homestead配置laravel开发环境后网页报错“No input file specified.”

    错误原因：虚拟机中nginx站点指定的文件夹与Homestead目录下Homestead.yaml文件里，sites指定的目录不同！！

    • 解决方法：
    1. 在虚拟机中查看并更改nginx站点指定文件夹的root(使之与Homestead.yaml文件里的sites指定目录相同)

    cd /etc/nginx/sites-available/
    vim homestead.test // 文件名可能不同,该目录下仅有一个文件

    2.重启虚拟机

    $ vagrant reload

    3.再次连接虚拟机后打开网页即可正常显示

    $ vagrant ssh
