git命令

2019.01.26 Xj Git  热度 ℃

常用

初始化仓库(进入项目根目录)

git init

显示工作目录和暂存区的状态

git status

将所有文件提交到索引仓库

git add .

克隆远程仓库到本地目录localdir

git clone http://github.xxx.git localdir

将索引仓库中的文件添加到git仓库

git commit -m “message”

将master分支推送到远程仓库master分支

git push origin master

切换分支

git checkout branchname

克隆仓库分支

git clone -b branch http://github.xxx.git

拉取远程分支

git pull <远程主机名> <远程分支名>:<本地分支名>
git pull origin master:brantest

将远程主机origin的master分支拉取过来，与本地的brantest分支合并。

git pull origin master

将远程origin主机的master分支拉取过来和本地的当前分支进行合并。

强制拉取远程master分支(覆盖本地)

git fetch origin master
git reset –hard origin/master

拉取远程跟踪的

git fetch -p

拉取远程分支并创建本地分支

git fetch origin newbranch:remotebranch

拉取远程分支remotebranch并创建本地分支newbranch

git checkout newbranch

查看所有远程分支

git branch -r

查看所有本地分支

git branch

删除本地分支branchname

git branch -D branchname

查看本地分支和远程分支的跟踪关系

git branch -vv

跟踪远程分支

git branch –set-upstream branch-name origin/branch-name

将本地branch-name分支追踪远程分支origin/branch-name

git branch -u origin/branch-name

设置当前分支跟踪远程分支origin/serverfix
将本地仓库追踪到远程仓库
  • 初始化本地仓库
  • 将本地master分支追踪到远程分支，远程分支别名origin，运行以下命令:

    git remote add origin https://github.com/jat0824/process_schedule.git

  • 将本地文件提交到仓库并推送到远程仓库

    git add .
    git commit -m “message”
    git push origin master

新建本地分支并切换到新分支，推到远程仓库

git checkout -b newbranch
git push origin newbranch

新建远程分支newbranch

git push origin newbranch:newbranch

删除远程分支branch

git push origin :branch

比较两个分支间所有变更的文件列表

git diff branch1 branch2 –stat // –stat 是显示文件列表, 否则是文件内容diff

查看本地具体文件改动细节

git diff filename

git diff ^M 的消除

git config –global core.whitespace cr-at-eol

分支重命名

git branch - m old_name new_name

合并分支

git checkout master
git merge branch1 // 出现冲突需要手动解决或使用合并工具
git mergetool // 使用合并工具解决冲突
