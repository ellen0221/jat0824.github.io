LNMP环境搭建--Ubuntu 16.04+php5.6

2019.04.18 Xj  热度 ℃

LNMP环境搭建

虚拟机：VMware

镜像：Ubuntu16.04 (ubuntu-16.04.6-desktop-amd64.iso)

Nginx安装：

sudo apt-get install nginx -y

  • 查看nginx版本

    nginx -v

  • 开启nginx服务

    sudo service nginx start

php5.6安装：

由于Ubuntu16.04自带php7，所以先要删除php7及其相关文件

sudo apt-get purge `dpkg -l | grep php| awk ‘{print $2}’ |tr “\n” “ “`

  • 添加PPA

    sudo apt-get install software-properties-common -y
    sudo add-apt-repository ppa:ondrej/php -y
    sudo apt-get update -y

  • 安装php5，6

    sudo apt-get install php5.6 -y

  • 安装扩展

    sudo apt-get install php5.6-mysql php5.6-common php5.6-curl php5.6-cli php5.6-mcrypt php5.6-mbstring php5.6-dom

  • 配置php5.6

    sudo vim /etc/php/5.6/fpm/php.ini
    找到cgi.fix_pathinfo选项，去掉注释;，然后将值设置为0:
    cgi.fix_pathinfo = 0;

mysql安装

sudo apt-get install mysql-server
sudo apt-get install mysql-client

nginx-php

sudo apt-get install php5.6-fpm
sudo service php5.6-fpm restart

  • 检查fpm运行状态

    netstat -ano |grep fpm

  • 修改nginx配置

    sudo vim /etc/nginx/sites-availabled/default

  • 重启nginx和fpm

    service php5.6-fpm restart
    service nginx restart

  • 测试nginx是否解析php
    在nginx配置文件中配置的root目录(我的是/var/www/html)中新建php信息页

    sudo chmod 777 /var/www/html //设置权限
    echo “<?php phpinfo(); “ > /var/www/html/phpinfo.php

访问localhost/phpinfo.php看到php信息页则成功

  • 遇到问题：访问phpinfo.php时显示空白页
    原因：nginx与php-fpm间的小bug，nginx无法正确的将php文件的地址传递给php-fpm解析，相当于php-fpm接收到了请求，但这个请求却只想一个不存在的文件，于是返回空结果。
    解决：修改nginx默认的fastcgi_params配置文件：/etc/nginx/fastcgi_params在文件的最后增加两行

最后重启nginx、fpm服务即可

安装git

sudo apt install git

  • 生成ssh

    git config –global user.name “yourname” // 与gitlab保持一致
    git config –global user.email “youremail@xx.com” // 与gitlab保持一致
    ssh-keygen -t rsa -C “youremail@xx.com” // 生成ssh密钥，一路回车。进入.ssh目录复制id_rsa.pub文件的内容，粘贴到gitlab新增的sshkey中。
